Что это

	virtualenv - виртуальное окружение, которое позволяет для каждого проекта хранить зависимости в отдельных окружениях, а глобально в системе.

	Когда мы запускаем виртуальный сервер, оно умеет обрабатывать http запросы и запускать python приложение.

Virtualenvwrapper

	Упрощает работу с виртуальным окружением

	Ubuntu

		pip install virtualenv

		pip install virtualenvwrapper

		Для установки в docker container (нужно писать в каждую новую консоль):

			apt-get python-pip

			pip install virtualenv

			pip install virtualenvwrapper

			export WORKON_HOME=~/.virtualenvs
			VIRTUALENVWRAPPER_PYTHON='/usr/bin/python3'
			source /usr/local/bin/virtualenvwrapper.sh

		workon - показывает все виртуальные окружения

		workon venv_name - выбирает и запускает виртуальное окружение (для того чтобы выбранное окружение привязалось к конкретному проекту, нужно перейти в директорию с проектом и выполнить эту команду)

		deactiavate - выход из выбранного виртуального окружения

		rmvirtualenv venv_name - удаление виртуального окружения

		which python3 - показывает где в систему питон

		mkvirtualenv -p /usr/bin/python3 venv_name - создание виртуального окружения для python3

		pip freeze list - показывает все установленные пакеты

		ls ~/.virtualenvs/ - место, где хранятся все виртуальные окружение, обычно директории с точками скрыты если смотреть не через консоль

	Windows

		pip install virtualenvwrapper-win

		Добавляем в переменные среды конкрентного пользователя (например Ilya) переменную WORKON_HOME и путь, где будут храниться venvs.

		lsvirtualenv - показывает все существующие

		mkvirtualenv name_env - создаёт виртуальное окружение

		rmvirtualenv name_env - удаляет окружение

		workon name_env - активирует окружение в той директории из которой произошёл вызов

		deactivate - выключает окружение
