В идеале для функциональных тестов схема такая:
- выбрать действие ("подтверждение карточки товара")
- описать use case (подтвердить можно через сохранение карточки товара, через action админки, через bin/manage shell)
- по этим use case-ам продумать тестовые кейсы (позитивные, негативные, etc; за один тест проверяем что-нибудь одно)
- создать детальное описание с действиями - комментами, что-то вроде такого:
    # зашел на страницу со списком карточек товаров
    # ввел код карточки 777 в строку поиска
    # нажал "Поиск"
    # увидел в списке найденных карточек искомую
    # выбрал ее галочкой
    # выбрал action админки "пометить как проверенные")
    # нажал "Выполнить"
    # проверил, что карточка действительно проверена (или нет, если тест негативный, тут проверяем сообщение/письмо/etc)
        (тут методом серого ящика - проверим date_formed_description и т.п.)

- пишем собственно действия под каждым комментом (для примера - на селениуме);
    для автоматизации можно использовать Selenium IDE (плагин к хрому/файерфоксу), там есть экспорт и копипаста записи действий в программный код на нужном языке программирования;
    проверяем, что код адекватен (не будет искать по какому-нибудь временному "id_94984984984"), правим, расставляем под комментами;

Если можно без проблем воспользоваться django client (нужен только логин, к примеру и все можно решить простым) - используем его;

Функционал добавления карточки товара

	После входа пользователь оказывается на главной странице по адресу http://0.0.0.0:8000/

	На панели управлени пользователь выбирает раздел "Карточки".

	Открывается выпадающий список и пользователь выбирает "Карточки товаров".

	Пользователь оказывается на странице со списком карточек товара по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь нажимает на кнопку "Добавить карточку товара".

	Пользователь оказывается на странице с формой для заполнения карточки товара по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь заполняет поле "Код 1с", "Вид-тип", "Описание", "Ответственный".

	Пользователь может нажать три кнопки на выбор "Сохранить и добавить другой объект", "Сохранить и продолжить редактирование", "Сохранить".

	После того, как пользователь нажал одну из кнопок он должен оказаться на странице со списком карточек товара по адресу http://0.0.0.0:8000/cards/productcard/ и сверху должно быть сообщение подсвеченное зеленым, например: Карточка товара "Неизвестно" был успешно добавлен.

	Примечания:

		Расписать варианты с валидацией
