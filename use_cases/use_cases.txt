Карточка товара

	Есть интернет магазины korallmicro и snr.

	У продукта представленного в этих магазинах может быть описание. Например произовдитель, модель, вес, размеры и т.д.

	Так же у продукта могут быть фотографии.

	Карточка товара это и есть тот раздел, который отвечает за создание описания и фотографий для продуктов. 

	Карточка товара создаётся и заполняется менеджерами.

	После создания карточки товара её нужно отредактировать, заполнив значения атрибутов.

		Для понимания всей картины читать разделы: атрибуты, шаблоны, вид-тип.

Функционал добавления карточки товара

	После входа пользователь оказывается на главной странице по адресу http://0.0.0.0:8000/

	На панели управлени пользователь выбирает раздел "Карточки".

	Открывается выпадающий список и пользователь выбирает "Карточки товаров".

	Пользователь оказывается на странице со списком карточек товара по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь нажимает на кнопку "Добавить карточку товара".

	Пользователь оказывается на странице с формой для заполнения карточки товара по адресу http://0.0.0.0:8000/cards/productcard/


	Пользователь заполняет все поля кроме "Вид-тип" и "Ответственный" и сохраняет.

	Пользователь оказывается на странице по адресу http://0.0.0.0:8000/cards/productcard/add/ 

	Пользователь видит ошибку в красной рамке с текстом, например: "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле с текстом "Обязательное поле".


	Пользователь заполняет все поля, вводит уже существующий код в поле "Код 1с" и нажимает сохранить.

	Пользователь оказывается на этой же странице и видит ошибку в красной рамке с текстом "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле "Код 1с" с текстом "Карточка товара с таким Код 1с уже существует."


	Пользователь заполняет поле "Код 1с", "Вид-тип", "Описание", "Ответственный" и сохраняет.

	Пользователь оказывается на странице отображения всех товара, по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь видит сообщение подсвеченное зеленым, например: Карточка товара "Неизвестно" был успешно добавлен.


	Вопросы:

		Что за атрибуты справа от поля "Описание" и как они работают?

			Все

			Да

		Ответственный

Атрибуты

	Атрибуты - это св-ва вида-типа. Например у вида-типа "Бумага-Копировальная" могут быть св-ва "Цвет бумаги", "Формат листов", "Плотность" и т.д.

	Атрибут можно добавить в шаблон.

	Атрибут можно добавить единожды, как в группу "Основные характеристики", так и в любые другие группы шаблона.

Функционал создания атрибута

	Пользователь выбирает на панели управления "Карточки".

	Пользователь видит выпадающий список.

	Пользователь выбирает "Атрибуты".

	Пользователь оказывается на странице отображения всех атрибутов, по адресу http://0.0.0.0:8000/cards/attributeoptions/

	Пользователь нажимает на кнопку "Добавить атрибут".

	Пользователь оказывается на странице добавления атрибута, по адресу /cards/attributeoptions/add/


	Пользователь заполняет все поля, кроме поля "Название атрибута" и нажимает сохранить.

	Пользователь видит в красной рамке сообщение об ошибке "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле с текстом "Обязательное поле."


	Пользователь заполняет все поля и нажимает кнопку сохранить.

	Пользователь оказывается на странице отображения всех атрибутов, по адресу http://0.0.0.0:8000/cards/attributeoptions/

	Пользователь видит сообщение в зеленой рамке с текстом, например: Атрибут "Атрибут Ильи" был успешно добавлен.

Шаблоны

	Шаблон - это сущность, которая имеет набор атрибутов.

	Атрибуты - это св-ва вида-типа. Например у вида-типа "Бумага-Копировальная" могут быть св-ва "Цвет бумаги", "Формат листов", "Плотность" и т.д.

	Шаблон можно привязать к виду-типу.


	У каждого шаблона есть дерево.

	В дерево шаблона можно добавлять группы.

	Группы могут быть вложены в другие группы. Например в группу "Основные характеристики" вложена группа "Общие". 

	Группа "Основные характеристики" является группой по умолчанию и всегда добавлена в дерево.


	Для каждой группы можно добавить и удалить атрибуты.

	Пользователь оказывается на странице для подтверждения создания, по адресу http://0.0.0.0:8000/cards/template/add/


	Пользователь нажимает "Нет, я передумал".

	Пользователь оказывается на странице отображения всех шаблонов по адресу http://0.0.0.0:8000/cards/template/

	Пользователь оказывается на странице редактирования созданного шаблона, например по адресу http://0.0.0.0:8000/cards/template/1528/change/

	Пользователь видит сообщение в зеленой рамке с текстом, например "Шаблон "Шаблон без привязки" был успешно добавлен. Ниже вы можете снова его отредактировать."

Функционал отображения шаблонов

	Пользователь выбирает на панели управления "Карточки".

	Пользователь видит выпадающий список.

	Пользователь выбирает "Шаблоны".

	Пользователь оказывается на странице отображения всех шаблонов, по адресу http://0.0.0.0:8000/cards/template/

	Пользователь видит:

		Поле для поиска с кнопкой "Найти".

		Поле "Действие" с выпадающим списком, кнопкой "Выполнить" и надписью "Выбрано 0 из n-числа" (в зависимости от кол-ва видов-типов на странице).

		Кнопку "Добавить шаблон".

		Таблицу с колонками: чекбокс (для выбора всех строк в таблице), шаблон.

		В строке таблицы пользователь видит:

			Чекбокс (для выбора конкретной строки)

			Шаблон (например Шаблоны для Широкоформатная-Бумага)

		Кнопки для пагинации с номерами страниц (если кол-во видов-типов больше чем кол-во разрешенных для вывода на одной странице).

Функционал редактирования шаблона

	Пользователь находится на странице отображения всех шаблонов, по адресу http://0.0.0.0:8000/cards/template/

	Пользователь нажимает на название в колонке "Шаблон".

	Пользователь оказывается на странице редактирования, например по адресу http://0.0.0.0:8000/cards/template/1527/change/?_changelist_filters=q%3D


	Пользователь видит:

		Дерево шаблона

		Внутри дерева шаблона группу "Основные характеристики"

		Кнопку "Добавить группу"

		Колонку со списком атрибутов

		Поле для поиска по атрибутам

		Ссылку "Все атрибуты"

		Кнопку "История"

		Collapse список, состоящий из:

			Шаблоны

			Шаблоны для поля описания

			Шаблоны для поля названия

			Шаблон представления дли Интернет-магазин "КораллМикро"

			Шаблон представления дли Интернет-магазин "Офисный Мир-Км"

			Шаблон представления дли Интернет-магазин "SNR".


	Пользователь перемещает курсор мыши влево и нажимает на группу, например на "Основные характеристики".

	Группа основные характеристики выделяется светло синим цветом.
	
	Пользователь перемещает курсор мыши вправо и нажимает на один из атрибутов из списка.

	Атрибут исчезает из списка и появляется в группе "Основные характеристики".

	У группы "Основные характеристики" слева появляется стрелочка.


	Если пользователь не нашел нужный атрибут из предложенных, он может установить курсов мыши в поле на всеми атрибутами, ввести нужное имя атрибута и произойдет поиск.

	Если по результатам поиска ничего не найдено, то пользователь увидит пустое пространство ниже поля для поиска.

	Если атрибут найден, то пользователь увидит его ниже поля поиск.


	Пользователь нажимает стрелочку слева от группы "Основные характеристики"

	Пользователь видит, выпадающий список атрибутов.

	Пользователь нажимает на один из атрибутов.

	Атрибут исчезает из группы "Основные характеристики" и появляется справа в списке всех атрибутов.


	Пользователь может добавить группу в дерево шаблона.

	Пользователь нажимает на кнопку "Добавить группу".

	В модальном окне браузера открывается страница для добавления группы, например по адресу http://0.0.0.0:8000/cards/templateentry/add/?_popup=1&template=1528

	Пользователь заполняет поля "Имя группы" и "Ориентация", после чего нажимает сохранить.

	Окно закрывается.

	Вопросы:

		Почему нет валидации при создании группы?

		Почему после сохранения она автоматом не добавляется в дерево, а нужно перезгружать систему?


	Пользователь может редактировать группу.

	Пользователь нажимает справа от названия группы на иконку "Гаечный ключ".

	В модальном окне браузера открывается страница для редактирования группы, например по адресу http://0.0.0.0:8000/cards/templateentry/5817/change/

	Пользователь редактирует поля "Имя группы" и/или "Ориентация" и нажимает сохранить.

	Вываливается ошибка "Page not found".

	Вопросы:

		Почему не возможно отредактировать?

Функционал удаления шаблона

	Пользователь нажимает кнопку выполнить.

	Пользователь оказывается на странице подтверждения удаления, по адресу http://0.0.0.0:8000/cards/template/


	Пользователь нажимает "Нет, отменить и вернуться к выбору".

	Пользователь оказывается на странице отображения всех шаблонов, по адресу http://0.0.0.0:8000/cards/template/


	Пользователь нажимает "Да, уверен".

	Пользователь оказывается на странице отображения всех шаблонов, по адресу http://0.0.0.0:8000/cards/template/

	Пользователь видит сообщение в зеленой рамке, например: Успешно удалены 1 Шаблон.

Вид-тип

	Оба этих термина (вид и тип) относятся к продукту.

	Вид - это, что за продукт. Например бумага.

	Тип - это, какого типа этот продукт. Например бумага может быть копировальная, фото бумага, копировальная и т.д.

	Исходя из вышесказанного получается например такой Вид-тип: Бумага-Копировальная.

Функционал создания вида-типа

	Уточнить у Глубокова.

Функционал отображения видов-типов

	Пользователь выбирает на панели уравления "Карточки".

	Открывается выпадающий список и пользователь выбирает "Вид-тип".

	Пользователь оказывается на странице со списком видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь видит:

		Поле для поиска с кнопкой "Найти".

		Поле "Действие" с выпадающим списком, кнопкой "Выполнить" и надписью "Выбрано 0 из n-числа" (в зависимости от кол-ва видов-типов на странице).

		Таблицу с колонками: чекбокс (для выбора всех строк в таблице), вид, тип, ответственный, карточек заполнено / всего.

		В строке таблицы пользователь видит:

			Чекбокс (для выбора конкретной строки)

			Вид (например Бумага)

			Тип (например Копировальная)

			Ответственный (может быть пустым, если не назначен)

			Карточек заполнено / всего (например 326 / 328 Список / Шаблон или Привязать Шаблон)

		Кнопки для пагинации с номерами страниц (если кол-во видов-типов больше чем кол-во разрешенных для вывода на одной странице).


		Пользователь может нажать на слово "Список" в колонке "Карточек заполнено / всего" и окажется на странице со списком карточек товара к которым привязан этот вид-тип.

		Пользователь может нажать на слово "Шаблон" в колонке "Карточек заполнено / всего" и окажется на странице с шаблоном, который привязан к этому виду-типу.

		Пользователь может нажать на название в колонке "Вид" и он окажется на странице редактирования вида-типа.

Функционал редактирования вида-типа
	
	Пользователь находится на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь нажимает на название в колонке "Вид".

	Пользователь оказывается на странице редактирования, например по адресу http://0.0.0.0:8000/cards/kind/1421/change/

	Пользователь видит поля: вид, тип, ответственный и кнопки: удалить, сохранить и продолжить редактирвоание, сохранить.


	Пользователь редактирует одно из обязательных полей (вид, тип), оставляет его пустым и сохраняет.

	Пользователь оказывается на той же странице и видит сообщение в красной рамке "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле с текстом "Обязательное поле."


	Пользователь заполняет все поля новыми значениями и нажимает сохранить.

	Пользователь оказывается на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь видит сообщение в зеленой рамке, например: Вид-тип "Автомагнитола-автомагнитола" был успешно изменен.

Функционал удаления вида-типа

	Пользователь оказывается на странице редактирования, например по адресу http://0.0.0.0:8000/cards/kind/1421/change/

Функционал привязки шаблона к виду-типу

	Пользователь находится на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь находит вид-тип у которого нет шаблона и в поле "Карточек заполнено / Всего" есть ссылка "Привязать шаблон".

	Пользователь нажимает на ссылку "Привязать шаблон".

	Пользователь оказывается на странице, где можно к виду-типу привязать шаблон, например по адресу http://0.0.0.0:8000/cards/property/add/?kind=848


	Пользователь оставляет поле "Шаблон" пустым и нажимает сохранить.

	Пользователь видит ошибку в красной рамке с текстом, например: "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле с текстом "Обязательное поле".


	Пользователь выбирает шаблон из выпадающего списка и нажимает сохранить.

	Пользователь оказывается на странице отображения всех свойств, по адресу http://0.0.0.0:8000/cards/property/

	Пользователь видит сообщение подсвеченное зеленым, например: Свойство "Свойства для Аккумулятор-Видео" был успешно добавлен.
