Вид-тип

	Оба этих термина (вид и тип) относятся к продукту.

	Вид - это, что за продукт. Например бумага.

	Тип - это, какого типа этот продукт. Например бумага может быть копировальная, фото бумага, копировальная и т.д.

	Исходя из вышесказанного получается например такой Вид-тип: Бумага-Копировальная.

Функционал создания вида-типа

	Уточнить у Глубокова.

Функционал отображения видов-типов

	Пользователь выбирает на панели уравления "Карточки".

	Открывается выпадающий список и пользователь выбирает "Вид-тип".

	Пользователь оказывается на странице со списком видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь видит:

		Поле для поиска с кнопкой "Найти".

		Поле "Действие" с выпадающим списком, кнопкой "Выполнить" и надписью "Выбрано 0 из n-числа" (в зависимости от кол-ва видов-типов на странице).

		Таблица с колонками: чекбокс (для выбора всех строк в таблице), вид, тип, ответственный, карточек заполнено / всего.

		В строке таблицы пользователь видит:

			Чекбокс (для выбора конкретной строки)

			Вид (например Бумага)

			Тип (например Копировальная)

			Ответственный (может быть пустым, если не назначен)

			Карточек заполнено / всего (например 326 / 328 Список / Шаблон или Привязать Шаблон)

		Пользователь может нажать на слово "Список" в колонке "Карточек заполнено / всего" и окажется на странице со списком карточек товара к которым привязан этот вид-тип.

		Пользователь может нажать на слово "Шаблон" в колонке "Карточек заполнено / всего" и окажется на странице с шаблоном, который привязан к этому виду-типу.

		Пользователь может нажать на название в колонке "Вид" и он окажется на странице редактирования вида-типа.

Функционал редактирования вида-типа
	
	Пользователь находится на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь нажимает на название в колонке "Вид".

	Пользователь оказывается на странице редактирования, например по адресу http://0.0.0.0:8000/cards/kind/1421/change/

	Пользователь видит поля: вид, тип, ответственный и кнопки: удалить, сохранить и продолжить редактирвоание, сохранить.

	Пользователь редактирует одно из обязательных полей (вид, тип), оставляет его пустым и сохраняет.

	Пользователь оказывается на той же странице и видит сообщение в красной рамке "Пожалуйста, исправьте ошибки ниже." + подсвеченное красным цветом поле с текстом "Обязательное поле."

	Пользователь заполняет все поля новыми значениями и нажимает сохранить.

	Пользователь оказывается на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь видит сообщение в зеленой рамке, например: Вид-тип "Автомагнитола-автомагнитола" был успешно изменен.

Функционал удаления вида-типа

	Пользователь находится на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь нажимает на название в колонке "Вид".

	Пользователь оказывается на странице редактирования, например по адресу http://0.0.0.0:8000/cards/kind/1421/change/

	Пользователь видит поля: вид, тип, ответственный и кнопки: удалить, сохранить и продолжить редактирвоание, сохранить.

	Пользователь нажимает кнопку удалить.

	Пользователь видит оказывается на странице подтверждения удаления, например по адресу http://0.0.0.0:8000/cards/kind/1421/delete/

	Пользователь нажимает кнопку "Да, я уверен".

	Пользователь оказывается на странице отображения всех видов-типов по адресу http://0.0.0.0:8000/cards/kind/

	Пользователь видит сообщение в зеленой рамке, например:Вид-тип "Автомагнитола-автомагнитола" был успешно удален.

Карточка товара

	Есть интернет магазины korallmicro и snr.

	У продукта представленного в этих магазинах может быть описание. Например произовдитель, модель, вес, размеры и т.д.

	Так же у продукта могут быть фотографии.

	Карточка товара это и есть тот раздел, который отвечает за создание описания и фотографий для продуктов. 

	Карточка товара создаётся и заполняется менеджерами.

Функционал добавления карточки товара

	После входа пользователь оказывается на главной странице по адресу http://0.0.0.0:8000/

	На панели управлени пользователь выбирает раздел "Карточки".

	Открывается выпадающий список и пользователь выбирает "Карточки товаров".

	Пользователь оказывается на странице со списком карточек товара по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь нажимает на кнопку "Добавить карточку товара".

	Пользователь оказывается на странице с формой для заполнения карточки товара по адресу http://0.0.0.0:8000/cards/productcard/

	Пользователь заполняет поле "Код 1с", "Вид-тип", "Описание", "Ответственный".

	Пользователь может нажать три кнопки на выбор "Сохранить и добавить другой объект", "Сохранить и продолжить редактирование", "Сохранить".

	После того, как пользователь нажал одну из кнопок он должен оказаться на странице со списком карточек товара по адресу http://0.0.0.0:8000/cards/productcard/ и сверху должно быть сообщение подсвеченное зеленым, например: Карточка товара "Неизвестно" был успешно добавлен.

	Если пользователь не заполнил обязательные поля ("Вид-тип", "Ответственный"), то после сохранения его должно вернуть на эту же страницу по адресу http://0.0.0.0:8000/cards/productcard/add/ и показать ошибку в красной рамке с текстом "Пожалуйста, исправьте ошибки ниже." + подсветить поля красным с текстом "Обязательное поле".

	Если пользователь ввел уже существующий "Код 1с", то после сохранения его должно вернуть на эту же страницу по адресу http://0.0.0.0:8000/cards/productcard/add/ и показать ошибку в красной рамке с текстом "Пожалуйста, исправьте ошибки ниже." + подсветить поле "Код 1с" красным с текстом "Карточка товара с таким Код 1с уже существует."
