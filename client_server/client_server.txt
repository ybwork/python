Взаимодействие клиент - сервер

	Клиент посылает запрос на сервер.

	Сервер обрабатывает запрос клиента и возвращает ответ клиенту.

Цикл запроса

	Если в адресную строку браузера введён поисковой запрос, например "самые красивые места мира", то:

		Браузер анализирует этот запрос и выдает в ответ список сайтов на которых может находится информация похожая на поисковой запрос.

	Если в адресную строку браузера введён адрес сайта, например https://www.facebook.com/, то:

		Браузер идет в файл hosts, который находится на вашем компьютере и ищет там имя сайта, в данном случае www.facebook.com

			В файле hosts хранятся имена сайтов и их ip адресса, например:

				582.33.96.77 www.mysite.com

		Если браузер находит в файле hosts нужное имя сайта, то он берет его ip адрес, без видимости пользователя подставляет подставляет его в адрессную строку браузера вместе с портом 80 (582.33.96.77:80) и делает запрос по этому ip адресу.

		Если браузер не находит имя сайта в файле hosts, то он делает запрос к dns-серверу, который находится на каком то другом компьютере в сети интернет.

			Адрес для обращения к dns-серверу хранится в настройках на вашем компьютере.

		Если в ответ на запрос браузера dns-сервер ничего не возвращает, то браузер сообщает пользователю о том, что такой сайт не найден.

		Если в ответ на запрос браузера dns-сервер нашел сайт, то он возвращает браузеру ip адрес этого сайта.

		После этого браузер без видимости пользователя подставляет ip адрес сайта в адрессную строку браузера вместе с портом 80 (582.33.96.77:80) и делает запрос по этому ip адресу.

		Дальше запрос обрабатывает веб-сервер.

			Если web сервер настроен на использование статического контента, то в ответ на запрос отпрвляется файл. Это может быть любой файл, в том числе и hmtl.

	Дальше запрос отправляется на порт 80.

	Порт 80 слушает nginx.

	Если запрос дошёл до web сервера, то это будет видно в логах. (access.log, error.log)

	Логи находятся /var/log/nginx

	Дальше управление берёт на себя web сервер.

	Если web сервер настроен на использование статического контента, то в ответ на запрос отпрвляется файл.

	Если web сервер настроен на использование динамического контента, то запрос отпрвляется в порт, который слушает application сервер.

	Application сервер обрабатывает запрос и запускает python приложение.
